✅ 4. OncePerRequestFilter — 서블릿 필터 구현 패턴
   Spring의 OncePerRequestFilter로 요청당 1회만 실행되는 필터를 만드는 방법.
   shouldNotFilter()로 특정 경로만 선택적으로 적용, doFilterInternal()에서 요청/응답 처리
   후 filterChain.doFilter()로 체인을 이어가는 구조.

✅ 5. Spring Security Filter Chain 구성 — addFilterBefore
   Spring Security의 필터 체인에 커스텀 필터를 삽입하는 방법. addFilterBefore(filter,
   UsernamePasswordAuthenticationFilter.class)의 의미와 필터 실행 순서. @Bean
   @Profile("prod") 메서드 파라미터로 Bean을 주입하는 Spring의 메서드 레벨 DI 패턴.

✅ 6. @ConfigurationProperties + @Validated — 설정값 검증과 fail-fast
   @NotBlank, @NotEmpty로 설정값을 앱 시작 시 검증하는 방법. ${ENV_VAR} (기본값 없음) vs
  ${ENV_VAR:default} (기본값 있음)의 차이, 그리고 운영 환경에서 필수 환경변수가 없을 때
   즉시 실패(fail-fast)시키는 이유.

보안 (Security)

6. X-Forwarded-For 헤더와 IP 스푸핑
   리버스 프록시(nginx 등) 뒤에서 클라이언트 실제 IP를 얻기 위해 X-Forwarded-For 헤더를
   사용하는 방법. 그러나 이 헤더는 클라이언트가 임의로 위조(스푸핑)할 수 있으므로, 직접
   연결된 피어(remoteAddr)가 신뢰된 프록시 목록에 있을 때만 X-Forwarded-For를 신뢰해야 하는
   이유.

7. API 키 인증 vs IP 기반 접근 제어의 차이
   API 키(X-Internal-Api-Key)는 "누가 호출하는지 식별"하지만, 네트워크 레벨 IP 제한은
   "어디서 호출하는지 차단"하는 추가 방어층임. 두 가지를 함께 적용하는
   Defense-in-Depth(다층 방어) 전략.

테스트

8. @WebMvcTest 슬라이스의 Bean 로딩 범위
   @WebMvcTest는 웹 레이어 Bean만 로드하는 슬라이스 테스트임. @Controller,
   Filter(@Component) 등은 포함되지만 @Service, @Repository, @ConfigurationProperties는
   포함되지 않음. @Component Filter가 @WebMvcTest에 포함될 때 그 의존 Bean이 없으면
   NoSuchBeanDefinitionException이 발생하는 원리와 해결 방법(@Component 제거 + @Profile
   메서드 파라미터 주입).

9. @ActiveProfiles("test")와 @Profile Bean의 관계
   테스트에서 @ActiveProfiles("test")를 사용하면 @Profile("prod") Bean은 생성되지 않음.
   @WebMvcTest에서 보안 필터 체인이 없어도 @AutoConfigureMockMvc(addFilters = false) +
   SecurityContextHolder 수동 설정으로 컨트롤러 테스트가 동작하는 원리.
